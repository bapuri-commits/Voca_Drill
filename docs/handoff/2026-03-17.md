# Voca_Drill Handoff — 2026-03-17

## 현재 상태

SyOps JWT 인증 연동 + user_id 계정 분리 + 로그인/홈 화면 + PDF 뷰어 + 객관식 퀴즈 UI 구현 완료. VPS 배포 완료 (`voca.syworkspace.cloud`).

## 작업 내용

- **JWT 인증**: `auth.py` 신규 — SyOps JWT 검증 (HS256, SYOPS_SECRET_KEY 공유), `get_current_user_id` Depends
- **user_id 분리**: WordProgress, LearningSession, DailyStats에 user_id 추가, Scheduler/DrillEngine/StatsTracker 쿼리에 user_id 필터
- **프론트 인증**: AuthContext (login/logout/authFetch), Login 페이지 (SyOps 로그인 API 호출), api.js에 authFetch 주입
- **새 홈 화면**: Dashboard 개편 — 스트릭, gradient Start Study 버튼, 오늘 통계, Level 분포, Quick Actions, Logout
- **PDF 뷰어**: 백엔드 `GET /api/pdf/{filename}` (path traversal 방지) + 프론트 iframe 뷰어
- **객관식 퀴즈 UI**: Study.jsx에 CardFlipQuiz/MultipleChoiceQuiz 분리, reverse도 같은 컴포넌트로 처리
- **Tailwind CSS 수정**: `@tailwindcss/postcss` + `postcss.config.js` 추가 (Vite 8 호환)
- **nginx**: `/syops-auth/` 프록시 추가 (SyOps CORS 우회)
- **배포**: Docker 재빌드, SYOPS_SECRET_KEY env 추가, nginx reload, PDF 디렉토리 생성

## 핵심 결정 & 맥락

- **독립 JWT 방식**: SyOps 프록시(BotTycoon 방식) 대신 JWT_SECRET 공유로 독립 인증
- **CORS 해결**: 프로덕션에서 nginx `/syops-auth/` 프록시, 로컬에서 직접 SyOps URL 호출 (import.meta.env.DEV 분기)
- **Word.progress relationship**: user_id 추가 후 `uselist=False`가 다중 사용자에서 충돌 가능 — 현재 2-3명 사용이라 보류
- **PDF는 서버 정적 파일**: 관리자가 VPS `/opt/data/voca-drill/pdf/`에 직접 업로드

설계 문서 통합 갱신 완료. 초록이 교재 분석, 영어 동의어 학습 핵심, Phase 0(데이터 확보) 선행 단계 추가. 코드 구현은 미착수 — 데이터 확보가 선행 과제.

## 작업 내용

- 이전 대화에서 논의한 학습 매커니즘(SM-2 + 라이트너 하이브리드, 4단계 평가, 세션 내 재출제)과 모바일 UX(풀스크린 카드, Thumb Zone, PWA) 의견 정리
- 사용자가 가져온 설계 문서(ADHD 친화, 다차원 인출 퀴즈, 게이미피케이션, NotebookLM 연동) 통합
- 초록이 교재 실제 페이지 사진 분석 → 데이터 구조 파악 (1단어 N뜻, 뜻마다 다른 동의어 세트)
- 핵심 발견: **초록이는 한국어 뜻 암기가 아닌 영어 동의어/유의어 암기가 학습의 본질**
- plan.md, architecture.md, features.md 3개 문서 전면 갱신

## 핵심 결정 & 맥락

- **타겟 정정**: 토플 특화 X → TOEFL/TOEIC 범용, 1차 데이터가 초록이
- **DB 설계 시점**: 데이터 확보 후 확정. OCR + NotebookLM 실데이터를 봐야 스키마를 잡을 수 있음
- **Word + WordMeaning 분리 검토**: 1단어 N뜻 구조를 정확히 담으려면 분리 필요 (초안, 확정 아님)
- **Phase 0 추가**: 데이터 확보가 구현 전 선행 과제로 명시
- **CLI 최소화**: 학습 세션 CLI는 만들지 않고 import/관리만. 바로 웹으로
- **카드 뒷면 설계**: 영어 동의어 목록이 중심, 한국어 뜻은 보조
- **퀴즈 설계**: Level 3 객관식이 "영어 동의어 보고 원래 단어 선택" (토플 직접 대비)
